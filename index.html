Easy — Recall & Definition
01 What is the primary purpose of vulnerability scanning? EASY
Correct: B. Vulnerability scanning builds an inventory of all devices on a network and creates a list of vulnerabilities that may affect each device. It is about identification and remediation, not exploitation.
02 What does CVE stand for? EASY
Correct: C. CVE stands for Common Vulnerabilities and Exposures — a publicly known list of vulnerabilities documented along with the software versions they affect.
03 What score range does CVSS use to rate vulnerability severity? EASY
Correct: C. CVSS scores range from 0 to 10, where 0 is least severe and 10 is the most severe.
04 An authenticated (credentialed) scan differs from an unauthenticated scan in what key way? EASY
Correct: B. An authenticated scan uses administrative credentials to access the device from within, providing a far more accurate vulnerability assessment used to drive remediation.
05 Which organisation currently owns and manages the CVSS framework? EASY
Correct: B. FIRST (Forum of Incident Response and Security Teams) owns and manages CVSS. NIAC originally introduced it in 2005.
06 Vulnerability scanners are NOT effective at detecting which type of vulnerability? EASY
Correct: C. Vulnerability scanners rely on databases of known vulnerabilities. They are not effective at detecting 0-day vulnerabilities because there are no known signatures for them yet.
07 A CVSS score of 9.5 falls into which severity category? EASY
Correct: C. Scores from 9.0–10.0 are rated Critical. High is 7.0–8.9, Medium is 4.0–6.9, and Low is 0.1–3.9.
08 What is the NCSC's definition of penetration testing? EASY
Correct: B. The NCSC defines pen testing as a method for gaining assurance in an IT system's security by attempting to breach it using the same tools and techniques as a real adversary.
09 Which of the following is a free/open-source vulnerability scanner? EASY
Correct: C. OpenVAS, Nmap, and Nikto are free/open-source tools. InsightVM, Nessus, and Qualys are paid commercial scanners.
10 What is "fingerprinting" in the context of vulnerability scanning? EASY
Correct: B. Fingerprinting identifies the operating system and software versions on a device by comparing known indicators of hardware and software responses.
Medium — Understanding & Application
11 Why does an unauthenticated scan tend to produce a high number of false positives? MEDIUM
Correct: B. Without credentials, the scanner cannot determine the exact version of software running. For example, detecting OpenSSH remotely without device access means the scanner may flag ALL OpenSSH vulnerabilities.
12 Which CVSS metric group can change over time as patches or workarounds become available? MEDIUM
Correct: C. Temporal metrics represent properties of a vulnerability that can change over time — such as the release of an official patch or changes in exploit code maturity.
13 How does the NCSC distinguish penetration testing from a routine vulnerability assessment process? MEDIUM
Correct: B. The NCSC states pen testing should be viewed as a method for gaining assurance in your organisation's vulnerability assessment and management processes — not as the primary way to find vulnerabilities.
14 What is the key advantage of agent-based (host-based) scanning over network-based scanning? MEDIUM
Correct: B. Agent-based scanning is especially useful for devices that are only intermittently connected to the company network, such as employee laptops at home. Network scanners can only scan devices available on the network at the time.
15 Which CVSS metric group is calculated by the end user (not the vendor) and is entirely optional? MEDIUM
Correct: C. The Environmental group score is calculated by the end user (not the vendor) and is optional. It refines the Base score to reflect the organisation's own environment and business context.
16 Why is it NOT advisable to allow a network-based vulnerability scanner full access to all ports across a firewall? MEDIUM
Correct: B. Allowing full access through a firewall removes important security segmentation. The recommended solution is to deploy secondary scanners on different sides of the firewall, reporting back to the primary console over a single encrypted TCP connection.
17 What is a major limitation of relying solely on publicly available CVSS scores when prioritising vulnerability remediation? MEDIUM
Correct: B. Public databases (NIST NVD) only show Base scores. Sole reliance on these neglects real-world exploit availability, existing mitigations, and the specific importance of the affected asset to the organisation.
18 What are the three ways a vulnerability scanner discovers what type of device is on the network? (Choose the set that lists all three) MEDIUM
Correct: B. The three fingerprinting techniques described are MAC address analysis, port fingerprinting (inferring OS from open ports), and banner grabbing (services responding with version info to Nmap probes).
19 In what key way does a penetration test differ from a vulnerability scan? MEDIUM
Correct: B. A vulnerability scan focuses on identifying weaknesses. A penetration test goes further — it involves identifying AND exploiting vulnerabilities with the goal of gaining a foothold in the network, just as a real adversary would.
20 Which type of vulnerability scan should be used against web applications to detect SQL injection or XSS? MEDIUM
Correct: C. Application scans crawl web application pages, follow links, and test input fields for vulnerabilities such as SQL injection and cross-site scripting (XSS) — the most critical vulnerability type for web applications.
Hard — Analysis & Evaluation
21 A TCP SYN scan is sometimes called a "stealth scan." Why does it avoid completing the TCP handshake, and what is the practical consequence of this for some firewalls? HARD
Correct: B. A SYN scan sends a SYN packet and considers any response as an active device indicator, without completing the handshake. Some firewalls respond with a TCP RESET, which might indicate a device behind the firewall — or indicate the firewall itself. Most scanners can be configured to skip ping requests or ignore TCP RESTs accordingly.
22 You run a credentialed network scan of Cisco routers and the results say they are "Apple iPhones" with 85% confidence. What is the MOST likely explanation? HARD
Correct: C. This is a classic credentialed scan failure. When credentials don't grant device access, the scanner falls back to external fingerprinting which gives only 85% confidence. The most likely cause is that the credentials have not provided access to the devices.
23 The NCSC compares a penetration test to a financial audit. What is the intended implication of this analogy? HARD
Correct: B. The analogy means your finance team tracks income/expenses daily (your internal processes); the audit verifies those processes are sufficient. Similarly, in an ideal world you should already know what the pen testers will find — they are verifying your internal vulnerability management is working.
24 CVSS Base scores are described as "mandatory" but the framework has a key limitation regarding business context. Which statement best captures this limitation? HARD
Correct: B. Base scores measure inherent, unchanging vulnerability properties. They do not account for business criticality, existing mitigating controls, or specific deployment context — these are captured in the Environmental score which organisations must calculate themselves.
25 Why does an organisation running a vulnerability scan against its own network sometimes need to skip the discovery phase entirely? HARD
Correct: B. Running discovery on a network you plan to scan with full credentials later means the scanner picks up inaccurate external fingerprints. It's better to skip discovery and let the credentialed scan collect accurate info from inside the device.
26 A vulnerability is detected with a CVSS Base score of 8.5, but the organisation's security team decides to treat it as lower priority than a vulnerability with a score of 5.0. Which factor most likely justifies this decision? HARD
Correct: B. Environmental context is crucial. CVSS Base scores don't reflect business criticality or existing controls. A high-score vuln on a non-critical isolated system may be lower priority than a medium-score vuln on a core payment system with no compensating controls.
27 A Wireless Intrusion Prevention System (WIPS) provides capabilities that go beyond basic vulnerability scanning for wireless threats. Which of the following is a WIPS capability NOT provided by standard vulnerability scanners? HARD
Correct: B. A WIPS can triangulate the exact physical location of rogue access points and automatically disable or isolate parts of the network as a preemptive action — capabilities that go far beyond what a standard vulnerability scanner can do for wireless threats.
28 PCI DSS compliance is affected by CVSS scores. Which statement about this relationship is accurate? HARD
Correct: B. Unpatched vulnerabilities with a CVSS score of 4.0 or higher (Medium and above) have an adverse effect on Payment Card Industry Data Security Standard (PCI DSS) compliance.
Scenario — Applied Thinking
29 SCENARIO: A security manager says "We had our pen test last month, so we're secure for the year." What is the most significant flaw in this reasoning? SCENARIO
Correct: B. The NCSC explicitly states that a pen test can only validate that systems are not vulnerable to known issues on the day of the test. Vulnerabilities could exist for long periods between tests if it's the only validation method.
30 SCENARIO: A company's network scanner keeps losing track of a laptop used by a remote salesperson because its IP address changes constantly. What is the BEST solution? SCENARIO
Correct: B. Agent-based scanning solves the dynamic environment problem. Each agent has its own unique ID, so even if the IP, hostname, or MAC address changes, scan results are always attributed to the correct device.
31 SCENARIO: Your scanner detects a vulnerability in Postgres databases whenever it finds port 5432 open. A developer says "we're safe — we patched it." What additional check should the scanner also perform to confirm remediation? SCENARIO
Correct: B. For database vulnerabilities, the scanner should run all related tests — not just patch detection. This includes standard CVE checks, default credential checks, and weak password checks. A patch alone may not fully remediate the risk.
32 SCENARIO: A coffee shop near your office is running a Wi-Fi access point with the exact same SSID as your corporate network. Employees are connecting to it. What type of threat is this, and would a standard network vulnerability scanner detect it? SCENARIO
Correct: B. This is a network impersonation / evil twin attack using a rogue access point with the same SSID. Standard vulnerability scanners cannot detect rogue APs. A Wireless Intrusion Prevention System (WIPS) is required to identify and respond to this threat.
33 SCENARIO: The remediation team wants to fix vulnerabilities in this order: newest CVEs first, then oldest. A senior analyst objects. What is the analyst's most likely valid concern? SCENARIO
Correct: B. The module notes that older vulnerabilities tend to have more resources available to help attackers exploit them — they are better documented. Remediation priority should weigh severity, ease of exploitation, age, and internal prioritisation together, not age alone.
34 SCENARIO: A small startup says "We'll just use Nmap and OpenVAS — they're free and cover enough." A consultant argues a commercial scanner is worth the investment. Which argument BEST supports the consultant's view? SCENARIO
Correct: B. Commercial scanners cover a far greater range (~500k vs ~50k for OpenVAS), have better accuracy with fewer false negatives, provide rich reporting and dashboards for remediation tracking, and include technical support — all significant advantages for a growing business.
35 SCENARIO: After a pen test, the report reveals a critical vulnerability. The remediation team wants to immediately apply a configuration change. The business continuity team objects. What does the module say should guide this decision? SCENARIO
Correct: B. The module states that throughout the remediation process, all changes must be carefully considered — it may also be necessary to facilitate some other business function. The remediation team should consult with other teams to assess risk versus necessity, and some vulnerabilities might be deemed acceptable risk regardless of severity.
Easy — Rapid7 Video Concepts
36 In the Rapid7 house analogy, what does "port scanning" compare to? EASY
Correct: C. Port scanning = using binoculars to see which doors/windows (ports) are open or closed. Vulnerability scanning = the home inspector assessing weaknesses. Pen testing = the ninja actually trying to break in.
37 What is the FIRST step of a penetration test according to the Rapid7 video? EASY
Correct: B. Before anything else, you establish a goal — for example "breach a credit card database." Everything that follows is directed by that goal.
38 What is "pivoting" in the context of a penetration test? EASY
Correct: B. Pivoting means using your initial foothold on one machine to jump to the next machine, then the next — moving deeper into the network from a single entry point like a phishing email.
Medium — Rapid7 Video Concepts
39 What is the difference between an internal and an external penetration test? MEDIUM
Correct: B. External = attacker attacking from outside over the internet. Internal = attacker already inside — could be a malicious insider, consultant with physical access, or someone who plugged a device into the network.
40 Which of the following is listed as a valid reason to conduct a penetration test? MEDIUM
Correct: B. The video lists several reasons: preventing data breaches, testing security controls, securing new systems before production, establishing a security baseline (e.g. for a new CISO), and meeting compliance requirements like PCI.
41 What should you collect BEFORE writing the final pen test report, and why? MEDIUM
Correct: B. Before writing the report, you must collect evidence — screenshots, downloaded files, captured password hashes — to prove to the client that you genuinely breached their systems, not just that you found potential weaknesses.
42 When is it generally more cost-effective to outsource a pen test rather than build an in-house capability? MEDIUM
Correct: B. If you only run one pen test a year, it's usually more cost-effective to outsource — the investment in training staff and purchasing tools isn't justified. In-house capability makes sense if you want more frequent testing (e.g. monthly).
Hard — Rapid7 Video Concepts
43 Why might an organisation choose NOT to tell its own security team that a pen test is taking place? HARD
Correct: B. One goal of pen testing is to test whether your security controls and team actually detect an attack. If the security team knows a test is happening, they will be on high alert — defeating the purpose of testing real detection and response capability.
44 When testing production environments, what is the recommended approach for exploit selection and why? HARD
Correct: B. Production environments must keep running. Tools like Metasploit rank exploits by reliability. Only "excellent" or "great" rated exploits should be used in production to ensure the test doesn't accidentally cause outages or data loss.
Scenario — Rapid7 Video Applied
45 SCENARIO: A pen tester gains access to a single laptop via a phishing email. The target — a credit card database — is on a different server deep inside the network. What technique allows them to reach it from that single laptop? SCENARIO
Correct: B. Pivoting is the technique of using an initial foothold (the phished laptop) to jump from machine to machine through the internal network — eventually reaching high-value targets like a database that would be inaccessible directly from the internet.
